Sales App - Offline Configurator

A worldwide manufacturer and distributor of agriculture equipment and solutions is pursuing excellence in its relationship with its dealers and customers by rethinking the whole customer experience – before, during and after a purchase. As part of DCX solutions/products, Sales App is an offline mobile/desktop application to help the its distributors with sales of their Machines, services and accessories to remote customers (farmers) integrated with CPQ(Configit) and CRM(Salesforce) with online-offline sync mechanism.

Xamarin Forms Cross Platform Application

One of the main projects I worked with this customer is on the Sales App which had limited functionality from the actual web application. The application demanded three main functionalities. It should support multiple platforms/divices, render 3D visualization and work offline.

I have chosen Xamarin Forms, an open source cross platform framework from Microsoft for building Android, iOS and Universal windows application using a common UI and shared code base making the development lean and cost effective.

3D visualization - three.js

Initially, the challenge was to get the lightweight data format to be used. My role was to extract LWG using automation from different CAD formats available in the PLM systems. Additionally, few customizations were done in Unity to produce FBX files (a popular 3D file format) for different configuration options.

Later, I adopted to use three.js a cross browser JavaScript library to assemble options, create scenes and perform animations in browser. Finally with the help of Xamarin WebView component I could able to integrate this functionality to mobile and windows apps. During the project, I also worked on a prototype on WebVR with three.js.

Offline/Online Sync

The biggest challenge we faced is to have an offline capability. Majorly, the offline-online sync mechanism since the data and the user base was too huge. We could able to resolve this easily using Couchbase Sync Gateway to synchronize meta data between offline peers and online server.

On the other hand, to synchronize digital assets (3D data, binary files, images, Configuration Packages) stored in Adobe DAM and AWS S3 buckets we used MuleSoft Orchestration. My role in both scenarios is setup the sync mechanism/orchestration to push/pull data to the SalesApp.
